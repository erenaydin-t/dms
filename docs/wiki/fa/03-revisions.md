# مدل بازنگری (بدون ابطال نسخه جاری)

از نسخه 2.3.0، بازنگری یک سند تحت کنترل **هرگز نسخه تأییدشده جاری را در ابتدای فرایند باطل نمی‌کند**. سازوکار قدیمی Cancel + Amend دیگر استفاده نمی‌شود.

## روند کار

```
QA-SOP-0001-1  (تأییدشده، فعال)          ← تا پایان فرایند، نسخه معتبر باقی می‌ماند …
      │  Create Revision (ایجاد بازنگری)
      ▼
QA-SOP-0001-2  (پیش‌نویس ← بررسی ← تأیید QA)   ← رکورد جداگانه، revision_of = …-1
      │  Approve as QA
      ▼
QA-SOP-0001-2  (تأییدشده، فعال)          ← اکنون نسخه معتبر است
QA-SOP-0001-1  (منسوخ، is_active = 0)    ← به‌صورت خودکار بازنشسته و برای ممیزی نگهداری می‌شود
```

1. روی سند **تأییدشده و فعال**، دکمه **Create Revision** را بزنید (نقش QA/DMS Manager) و «دلیل تغییر» الزامی را وارد کنید.
2. یک **رکورد پیش‌نویس جدید** با بخش نسخه بعدی ساخته می‌شود که با فیلد `revision_of` به سند قبلی متصل است. این پیش‌نویس **بدون پیوست** شروع می‌شود — فایل بازنگری‌شده را بارگذاری کنید. فایل تحت کنترل نسخه قبلی هرگز بازاستفاده یا جایگزین نمی‌شود.
3. پیش‌نویس، گردش‌کار کامل بررسی را طی می‌کند. در تمام این مدت، نسخه قبلی **تأییدشده، فعال و معتبر** می‌ماند و PDF نسخه کنترل‌شده آن همچنان واترمارک `CONTROLLED COPY` دارد.
4. با تأیید نهایی QA:
   - نسخه جدید معتبر می‌شود و شماره نسخه رسمی افزایش می‌یابد (`version_number` = قبلی + 1)
   - نسخه قبلی به‌صورت خودکار **منسوخ** می‌شود (`is_active = 0` اما docstatus = 1 می‌ماند — رکورد ممیزی غیرقابل تغییر)
   - ارجاع‌های سایر مستندات به‌صورت خودکار به نسخه جدید منتقل می‌شوند

## لغو بازنگری

در هر مرحله پیش از تأیید، از منوی Actions گزینه **Cancel Revision** را بزنید:

- پیش‌نویس به وضعیت پایانی **Revision Cancelled** می‌رود و **در پایگاه داده نگهداری می‌شود** (حذف آن برای حفظ ردیابی ممیزی مسدود است)
- نسخه قبلی کاملاً دست‌نخورده می‌ماند — همچنان تأییدشده و معتبر، با شماره نسخه بدون تغییر
- بلافاصله می‌توان بازنگری تازه‌ای آغاز کرد؛ نام آن **بخش بعدی** را می‌گیرد (بازنگری لغوشده `…-2` هرگز با تلاش بعدی `…-3` تداخل ندارد)

## قواعد کنترلی (اجرا در سمت سرور)

- فقط **نسخه معتبر جاری** (تأییدشده + فعال + ثبت‌شده) قابل بازنگری است.
- برای هر سند **فقط یک بازنگری باز** مجاز است — درخواست دوم تا زمان بسته‌شدن اولی رد می‌شود (بازنگری‌های لغوشده مانع نیستند).
- سندی که قبلاً جایگزین شده، هرگز نمی‌تواند جانشین موازی دیگری بگیرد.
- «دلیل تغییر» برای هر بازنگری الزامی است.

## مثال عملی (از اجرای آزمون پذیرش)

| گام | رکورد | وضعیت پس از گام |
|---|---|---|
| ایجاد + تأیید | `QA-SOP-0001-1` نسخه 1 | تأییدشده، فعال |
| بازنگری + تأیید | `QA-SOP-0001-2` نسخه 2 | تأییدشده، فعال؛ `…-1` منسوخ |
| بازنگری، سپس **لغو بازنگری** | `QA-SOP-0001-3` | بازنگری لغوشده (نگهداری‌شده)؛ `…-2` همچنان فعال |
| بازنگری مجدد + تأیید | `QA-SOP-0001-4` نسخه 3 | تأییدشده، فعال؛ `…-2` منسوخ |

به تفاوت بخش نام (`-4`) و شماره نسخه (`3`) توجه کنید: نام همه تلاش‌ها را می‌شمارد، شماره نسخه رسمی فقط تأییدها را.
