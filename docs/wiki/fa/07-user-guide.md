# راهنمای کاربری — گام‌به‌گام

## ۱. ایجاد سند تحت کنترل جدید

1. **GMP Document ← New**
2. هر دو عنوان (فارسی و انگلیسی)، نوع سند، واحد (باید مخفف داشته باشد)، اثر GMP و دوره اعتبار را پر کنید.
3. **Word Template** را انتخاب و **بررسی‌کننده** و **تأییدکننده QA** را تعیین کنید — هر دو باید تصویر امضا روی رکورد Employee داشته باشند (در غیر این صورت همان لحظه هشدار می‌گیرید).
4. فایل **docx** مبدأ حاوی تگ‌های قالب را پیوست کنید (صفحه *قالب‌ها*).
5. ذخیره کنید. سند شناسه دائمی می‌گیرد (مثل `QA-SOP-0001-1`)؛ فایل به همان نام تغییر می‌یابد و هش یکپارچگی ذخیره می‌شود.
6. **Actions ← Submit for Review** (فقط شما به‌عنوان تهیه‌کننده می‌توانید).
7. بررسی‌کننده وظیفه می‌گیرد ← **Actions ← Approve as Reviewer** (یا *Request Revision* برای برگشت).
8. تأییدکننده QA ← **Actions ← Approve as QA**. سند ثبت، رندر، امضا و مهر می‌شود و **نسخه معتبر** می‌گردد (نشانگر سبز *Controlled*).

## ۲. دانلود PDF

روی هر سند تأییدشده ← منوی **Get PDF**:

- **Controlled Copy** — توزیع رسمی با واترمارک وضعیت
- **Uncontrolled Copy** — واترمارک `UNCONTROLLED COPY` + پاورقی تاریخ شمسی چاپ
- **Plain** — بدون واترمارک، فقط پاورقی تاریخ شمسی چاپ

وضعیت واقعی همیشه نمایان است: سند منسوخ واترمارک `OBSOLETE` و سند با تاریخ اجرای آینده `NOT YET EFFECTIVE` می‌گیرد — در **همه** انواع.

## ۳. بازنگری سند

1. نسخه معتبر جاری را باز کنید ← **Create Revision** ← «دلیل تغییر» را بنویسید.
2. به پیش‌نویس جدید (`…-2`) منتقل می‌شوید. **سند اصلی معتبر و فعال می‌ماند** — هنوز هیچ اتفاقی برای آن نیفتاده است.
3. فایل docx بازنگری‌شده را بارگذاری کنید (فایل نسخه قبلی منتقل نمی‌شود)، فیلدها را تنظیم و همان گردش‌کار بررسی را طی کنید.
4. با تأیید QA، نسخه جدید معتبر می‌شود و نسخه قبلی خودکار **منسوخ** می‌گردد. ارجاع‌های سایر مستندات هم خودکار منتقل می‌شوند.

برای انصراف در هر مرحله پیش از تأیید: **Actions ← Cancel Revision**. پیش‌نویس برای ممیزی نگه داشته می‌شود (وضعیت *Revision Cancelled*) و سند اصلی معتبر می‌ماند. بلافاصله می‌توانید بازنگری تازه‌ای شروع کنید.

مجاز نیست: بازنگری نسخه منسوخ یا در انتظار اجرا، دو بازنگری هم‌زمان روی یک سند، حذف بازنگری لغوشده.

## ۴. زمان‌بندی یا عقب‌بردن تاریخ اجرا

1. روی پیش‌نویس، گزینه **Edit Effective Date** را روشن کنید (مختص QA/DMS Manager) و تاریخ را انتخاب کنید.
2. **تاریخ آینده:** پس از تأیید QA، سند نشانگر نارنجی *Effective DD-MM-YYYY* و واترمارک `NOT YET EFFECTIVE` می‌گیرد؛ نسخه جاری فعلاً معتبر می‌ماند. در موعد مقرر، زمان‌بند روزانه جابه‌جایی را خودکار انجام می‌دهد.
3. **تاریخ گذشته:** سند بلافاصله معتبر می‌شود؛ انقضا (+۲/۳/۵ سال) و بازنگری بعدی (−۱ ماه) از همان تاریخ محاسبه می‌شوند.
4. با خاموش‌کردن گزینه، کنترل تاریخ به سیستم برمی‌گردد (تاریخ تأیید).

## ۵. بازنشستگی سند بدون جایگزین

سند تأییدشده را Cancel کنید. سند منسوخ می‌شود، رکوردش می‌ماند و PDFهایش واترمارک `OBSOLETE` می‌گیرند. زمان‌بند انقضا نیز پس از `expiry_date` همین کار را خودکار می‌کند.

## پیام‌های متداول

| پیام | معنا |
|---|---|
| *Reason for Change is mandatory when revising…* | هر بازنگری به دلیل مستند نیاز دارد |
| *A revision of X is already in progress: Y* | ابتدا بازنگری باز را تمام یا لغو کنید |
| *Only the current effective version can be revised* | نسخه منسوخ/در انتظار را باز کرده‌اید — نسخه فعال را بازنگری کنید |
| *…cannot be used until a signature is configured* | بررسی‌کننده/تأییدکننده تعیین‌شده تصویر امضا ندارد |
| *'Edit Effective Date' is enabled but no Effective Date is set* | تاریخ را وارد یا گزینه را خاموش کنید |
| *This document cannot be submitted directly* | از منوی Actions گردش‌کار استفاده کنید، نه دکمه Submit |
| *Cancelled revisions are retained for audit…* | حذف بازنگری لغوشده عمداً مسدود است |
